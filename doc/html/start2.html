Running a Program

Now try to run the program hello.gb (you didn't expect this, did you? ;-)

gbeta hello

The program looks like this: 

-- universe:descriptor --
// FILE hello.gb
{
  'Hello, world!\n' | stdio
}

The output from the execution is just the text "Hello, world!", but if we add option -v it gets quite verbose: 

GBETA version 1.9.11, copyright (C) 1997-2011 Erik Ernst.

For details please see the file COPYING.

Setting option v: Print startup messages etc.

Main source code file: "hello"

------------------------------ Analyze program

------------------------------ Execute primary object
Hello, world!

The verbose variant may be useful because it provides a greater amount of information about what is going on (in particular if you want to see a short description of each option selected), but the default behavior is to avoid too much noise, just like most UNIX style utilities.

Also note that the gbeta tool by default works as a combination of a compiler and a virtual machine: It will compile the program given as a command line argument (as well as the files included directly or indirectly) to bytecodes, and then the built-in virtual machine will execute the bytecodes. You may use option -g to request bytecode generation only, but this will not be necessary for this tutorial so we will not go into details.

The next subject is interactive execution, not unlike a debugging session.
