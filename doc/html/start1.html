Checking the installation

If you have not already installed gbeta, then please do that before you proceed here. In this section it is assumed that you have installed in your $HOME directory. If not, you must adjust some paths accordingly.

Checking that gbeta is there

Generally the experiments in this section take place in the start directory:

cd $HOME/gbeta-1.9.11/examples/start

First try to execute gbeta without file arguments, like this:

gbeta -v

If the gbeta invocation script is in the PATH or otherwise reachable, and if gbeta itself is present for the current platform, and if it can find the grammar specifications, then you will get a startup message beginning like this:

GBETA version 1.9.11, copyright (C) 1997-2011 Erik Ernst.
This is free software, and you are welcome...

If you can get this message, the installation works as expected. If you run gbeta entirely without arguments you will get a listing of the available options along with a brief explanation. This is a bit unwieldy if we just want to check that gbeta is installed, but it may come in handy later.

Next, we'll run an actual program!
