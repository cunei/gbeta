Compiling gbeta, Creating Variants

If you have downloaded the soure code, and you have access to a Mjolner BETA System compiler, you can compile gbeta, and hence you can perform language design experiments with it. Since I'd enjoy cooperating about such endeavors, don't hesitate to ask me if you experience problems in doing so!

Note that some special considerations are needed in connection with compilation on Windows platforms.

Assuming you have installed gbeta in your home directory, you can compile gbeta like this:

cd $HOME/gbeta-1.9.11/src/main
beta -lq gbeta

This will take some time. The binary created by this compilation must be moved to the bin directory of your gbeta installation, and it must be named in a way which shows what platform it belongs to, i.e. you must execute one of the following commands:

On Linux:

mv gbeta ../../bin/gbeta-1.9.11-i386-linux-elf-bin

On Sun:

mv gbeta ../../bin/gbeta-1.9.11-sparc-sun-solaris-bin

That's it! Now, assuming that you have adjusted your path to include the gbeta executable as explained under installation, you can try out your new gbeta, to see if your installation is basically sound.

Special Considerations on Windows

Note that the Windows version of the source code can not co-exist with versions for other platforms since Windows does not distinguish between upper and lower case letters in filenames, and generated filenames (sitting inside some generated files) will consequently have modified case, and then the other platforms cannot use those files. To avoid these problems, just install the Windows version in its own directory or on another disk, keeping the files entirely separate. This will typically be the case for most installations under Windows, because a Windows disk is often not shared with computers running other operating systems.

Other than that, the approach is similar to the other platforms:

cd c:\gbeta-1.9.11\src\main
beta -lq gbeta

and:

mv gbeta ..\..\bin\gbetabin.exe

Creating Variants

Of course there could be myriads of different ways to modify gbeta. If you want to experiment with the grammar of the language, you need the dogram tool which is delivered with the Mjolner BETA System. This tool will generate grammar tables from your customized version of the involved grammars, and gbeta cannot use a changed grammar without having these generated files. The script $GBETA_ROOT/bin/dogrammar will help you getting this right.

Changing the grammar of the language is of course potentially a very deep change which requires a lot of changes to the static analysis and run-time system of gbeta. Don't hesitate to tell me about your adventures if you are doing such things!
